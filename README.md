# Скрипты для Adobe Premiere Pro

Этот репозиторий предназначен для заметок и примеров по созданию пользовательских скриптов для Adobe Premiere Pro на базе ExtendScript (диалект JavaScript, используемый в приложениях Adobe) и CEP-панелей. Ниже — краткое руководство, которое поможет начать работу.

## Что потребуется
- **Adobe Premiere Pro** (Creative Cloud, не старше CC 2019).
- **ExtendScript Toolkit** (устаревший, но всё ещё полезный) или расширение **ExtendScript Debugger** для VS Code.
- **Базовые знания JavaScript**: синтаксис, работа с объектами и функциями.
- **Документация SDK**: [Premiere Pro Scripting Guide](https://ppro-scripting.docsforadobe.dev/) и [ExtendScript Toolkit Guide](https://extendscript.docsforadobe.dev/).

## Установка окружения
1. Установите Adobe Premiere Pro через Creative Cloud.
2. Поставьте VS Code и расширение **ExtendScript Debugger** (Adobe), чтобы иметь подсветку синтаксиса и отладку.
3. Создайте папку `Scripts` рядом с этим README для хранения `.jsx` файлов (ExtendScript).
4. При необходимости установите **Adobe CEP Samples** — они демонстрируют, как создавать панели на HTML/JS.

## Первый скрипт
1. Создайте файл `hello.jsx` и разместите в нём код:
   ```jsx
   alert("Привет, Premiere Pro!");
   ```
2. Скопируйте файл в папку `C:/Program Files/Adobe/Adobe Premiere Pro <версия>/Scripts/` (Windows) или `/Applications/Adobe Premiere Pro <версия>/Scripts/` (macOS).
3. В Premiere Pro откройте **File → Scripts → Run Script File…** и выберите файл.
4. Увидите всплывающее окно с сообщением.

## Основные объекты API
- `app.project`: доступ к проекту, последовательностям, клипам.
- `ProjectItem`, `Sequence`, `Track`, `TrackItem`: элементы таймлайна и медиа.
- `qe`: внутренний объект (QuickTime Engine) с расширенными возможностями, доступен через `qe.project`. Используйте осторожно — не документирован официально.

## Советы по разработке
- Всегда делайте бэкап проекта перед запуском новых скриптов.
- Изучайте примеры в папке `Sample Projects` из Adobe — там есть готовые `.jsx` файлы.
- Используйте `$.writeln()` для логирования и просматривайте логи в ExtendScript Toolkit или консоли VS Code.
- Разбейте скрипты на функции, чтобы облегчить повторное использование.

## CEP-панели (HTML/JS)
Если вам нужен сложный UI:
1. Установите [Adobe CEP](https://github.com/Adobe-CEP/CEP-Resources).
2. Создайте HTML/JS-панель, которая взаимодействует с ExtendScript через `CSInterface`.
3. Соберите и установите панель в каталоге `CEP/extensions`.

## Панель GARSUS Prod Sync
Расширение `com.garsus.sync.panel` располагается в каталоге `%APPDATA%\Adobe\CEP\extensions\com.garsus.sync.panel\` и поддерживает Premiere Pro 23.0–23.9 (Windows 11). Манифест находится в `CSXS/manifest.xml`.

### Установка
1. Скопируйте содержимое репозитория в `%APPDATA%\Adobe\CEP\extensions\com.garsus.sync.panel\`.
2. Включите режим отладки CEP, добавив параметр `HKCU\Software\Adobe\CSXS.11\PlayerDebugMode` со значением `"1"` (если ещё не включен).
3. Запустите Premiere Pro и откройте панель через **Window → Extensions → GARSUS Prod**.

### UI
Панель содержит:
- глобальную настройку `Default Replay length (sec)` (значение ≥ 1, по умолчанию 600);
- чекбокс `Mute V1–V3/A1–A9 перед вставкой` для временного отключения трёх видеодорожек и девяти аудиодорожек перед вставкой;
- три блока для дорожек V1, V2 и V3 со кнопками синхронизации, флагом «Начало» и полем «Сдвиг времени (±HH:MM:SS)»;
- лог в нижней части панели для отображения статуса и ошибок.

### Логика синхронизации (`jsx/host.jsx`)
- Собирает все клипы на выбранной видеодорожке активной секвенции и пытается распознать время из имени файла по шаблону `YYYY-MM-DD HH-MM-SS[...].ext` или `YYYY-MM-DD_HH-MM-SS[...].ext`.
- Для каждой записи рассчитывает старт клипа с учётом опции «Начало/Конец», индивидуального сдвига и при необходимости использует длительность из медиа или фоллбек (`Default Replay length` / 1 секунда).
- Обрезает клипы, если старт < 0, перекрывает предыдущие вставки или выходит за пределы суток (24 часа).
- Перед вставкой гарантирует наличие минимум трёх видеодорожек и девяти аудиодорожек, при возможности создавая недостающие дорожки через QE API.
- Вставляет новые экземпляры клипов на выбранную видеодорожку, раскладывает связанный звук по диапазонам A1–A3, A7–A9 или A4–A6, очищает In/Out и, при успехе, переносит CTI на позицию первого вставленного клипа.
- В лог записывает сводку `OK[VX] placed=… skippedInside=… trimmedHeadSec=… clampedToZeroSec=… errors=… noParse=…` и дополнительные предупреждения (`DURATION UNKNOWN -> using …`, `WARN_NEED_TRACKS`, `INSERT FAIL …`, `HOST ERROR …`).

### UI-скрипт (`app/js/main.js`)
- Загружает `host.jsx`, валидирует поля формы, преобразует сдвиг времени в секунды и вызывает `garsus_syncTrack` с параметрами дорожки.
- Отображает статус выполнения и вывод host-ответы в лог панели.
